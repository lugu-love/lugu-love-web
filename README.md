# lugu-love-web

Lugu Lake Love & Civilization Web System (MVP)

## 开发规则

1. 本项目唯一代码仓库：lugu-love/lugu-love-web
2. 默认修改 index.html，除非明确要求创建新文件。
3. 每次只完成一个任务。
4. 修改完成后直接 Commit 到 main。
5. 未经允许，不修改其它功能。
6. 修改前必须先读取当前仓库代码。
7. 禁止引用本地路径。
8. 禁止引用旧聊天内容。
9. Commit 信息格式：
   - feat: 功能名称
   - fix: 修复内容
   - style: 样式调整

## 默认设计规则：手机竖屏优先

1. 所有新页面、界面、按钮、动画、栏目和交互，默认先按手机竖屏设计，再考虑电脑端兼容。
2. 重点适配手机宽度：360px、375px、390px、414px、430px。
3. 首屏优先使用 `100svh`，必要时兼容 `100dvh`。
4. 核心页面默认采用手机竖屏结构；只有明确要求电脑端或横屏版时，才单独设计电脑横屏。
5. 电脑端只作为预览和兼容环境。沉浸式页面、地球首页、泸沽湖母主页等在电脑端中间显示手机竖屏页面，最大宽度控制在 430px 左右，左右使用深色或与页面协调的留白。
6. 手机端主视觉图不拉伸、不严重裁切人物、不破坏原始竖屏构图，优先保证人物、湖面、天空、标题留白区域完整。
7. 开场视频和人物动画默认在手机竖屏容器内播放，不以整个电脑浏览器为全屏基准，不使用 `width: 100vw` 和 `height: 100vh` 强行铺满电脑横屏。
8. 页面不得出现横向滚动，文字不得超出屏幕，按钮点击区域不小于 44px，顶部元素不重叠，底部导航必须适配手机安全区域，并使用 `env(safe-area-inset-*)`。
9. 每次任务顺序：先完成手机端，检查 390px 宽度效果，再做电脑端兼容，最后才考虑平板和其他尺寸。
10. 除非明确要求，不优先设计电脑横屏版，不重构整个项目，不同时修改多个页面，不主动检查所有页面，不推送上线，不部署远程环境。
